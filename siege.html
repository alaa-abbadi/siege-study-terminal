frequently asked

Does SIEGE generate practice questions with AI?
No. This is the core principle. AI-generated questions are unreliable and don't match real CAIE mark schemes. SIEGE only references real past papers (with real paper codes like 0620/42/O/N/22) and links to public mirrors like gceguide.com.

Where is my data stored?
Entirely in your browser's localStorage. Nothing is sent to any server except your Groq API calls (which only contain the chat messages you send). You can export everything as JSON, encrypted or plain.

Can I add subjects not in the catalog?
Yes — see CUSTOMIZATION.md. The syllabus is editable JSON in the source. Add your own topics, paper structure, and codes.

Why brutalist?
Because every other study app looks the same. SIEGE makes opening it feel different — like a command terminal, not another Notion clone. The aesthetic is part of the discipline.

Which Groq model should I use?
Default gpt-oss-120b is best for reasoning (explanations, mark scheme grading). Switch to llama-3.1-8b-instant if you want sub-second responses for the daily plan.

Does it work offline?
Mostly. Mission generation has a rule-based fallback. AI chat needs internet + Groq key. Everything else (topics, papers, notes, pomodoro, stats) works offline.

google calendar setup

  1. go to console.cloud.google.com → new project
  2. enable Google Calendar API
  3. OAuth consent screen → External → add your email
  4. Credentials → OAuth 2.0 Client ID → Web Application
  5. add your siege.html's file:// URI or hosting URL to "Authorized JavaScript origins"
  6. copy the Client ID → paste into SIEGE Settings

notion integration setup

Notion blocks direct browser API calls (CORS). You need a Cloudflare Worker proxy:

// Cloudflare Worker for Notion CORS proxy
export default {
  async fetch(request) {
    const url = new URL(request.url);
    const notionUrl = 'https://api.notion.com' + url.pathname + url.search;
    const response = await fetch(notionUrl, {
      method: request.method,
      headers: request.headers,
      body: request.body
    });
    const headers = new Headers(response.headers);
    headers.set('Access-Control-Allow-Origin', '*');
    return new Response(response.body, { status: response.status, headers });
  }
}

Deploy this worker, then paste its URL into Settings → Notion Proxy URL.
